# 2. Vì sao phương án B tối ưu nhất?

## 2.1. Áp dụng Level-based Explanation

Phương án B yêu cầu giải thích Spring AOP theo 2 cấp độ:

```text
Cấp độ 1: Cho người mới bắt đầu, dùng ví dụ đời sống.
Cấp độ 2: Cho Senior Developer, dùng thuật ngữ chuyên sâu.
```

Đây là điểm rất quan trọng khi học kiến thức kỹ thuật nâng cao. Với người mới, các khái niệm như `Aspect`, `Pointcut`, `Advice` khá khó hiểu. Nếu giải thích ngay bằng thuật ngữ hàn lâm, người học dễ bị rối.

Ví dụ có thể hiểu đơn giản:

```text
AOP giống như một trạm kiểm soát giao thông.
Mọi xe đi qua một tuyến đường đều được kiểm tra tự động,
mà không cần mỗi tài xế tự viết quy trình kiểm tra.
```

Trong Spring Boot, AOP cũng tương tự: ta có thể tự động thêm logic ghi log, kiểm tra quyền, đo thời gian thực thi vào nhiều method mà không cần viết lặp lại trong từng service.

## 2.2. Áp dụng Comparative Analysis

Phương án B yêu cầu so sánh giữa:

```text
AOP - Aspect-Oriented Programming
OOP - Object-Oriented Programming
```

Việc so sánh giúp người học hiểu AOP không thay thế OOP, mà bổ sung cho OOP.

| Tiêu chí          | OOP                                             | AOP                                         |
| ----------------- | ----------------------------------------------- | ------------------------------------------- |
| Mục tiêu chính    | Tổ chức code theo object/class                  | Tách logic cắt ngang ra khỏi business logic |
| Đơn vị chính      | Class, Object, Method                           | Aspect, Pointcut, Advice                    |
| Phù hợp để xử lý  | Nghiệp vụ chính của hệ thống                    | Logging, Security, Transaction, Monitoring  |
| Vấn đề giải quyết | Mô hình hóa đối tượng                           | Tránh lặp code ở nhiều nơi                  |
| Ví dụ             | `UserService`, `OrderService`, `PaymentService` | Ghi log tự động cho toàn bộ Service         |

Nhờ bảng so sánh này, người học hiểu rõ AOP dùng để xử lý các vấn đề lặp lại ở nhiều tầng của hệ thống, còn gọi là **Cross-cutting Concerns**.

## 4.2. Aspect ghi log thời gian thực thi Service

```java
package com.example.demo.aspect;

import lombok.extern.slf4j.Slf4j;
import org.aspectj.lang.JoinPoint;
import org.aspectj.lang.annotation.After;
import org.aspectj.lang.annotation.Aspect;
import org.aspectj.lang.annotation.Before;
import org.springframework.stereotype.Component;

@Slf4j
@Aspect
@Component
public class ServiceLoggingAspect {

    private final ThreadLocal<Long> startTime = new ThreadLocal<>();

    /*
     * Pointcut:
     * execution(* com.example.demo.service..*(..))
     *
     * Ý nghĩa:
     * - execution: bắt các method được thực thi.
     * - *: method có kiểu trả về bất kỳ.
     * - com.example.demo.service..*: áp dụng cho tất cả class trong package service và các package con.
     * - *(..): áp dụng cho tất cả method, với số lượng tham số bất kỳ.
     */

    @Before("execution(* com.example.demo.service..*(..))")
    public void logBeforeServiceMethod(JoinPoint joinPoint) {
        startTime.set(System.currentTimeMillis());

        log.info(
                "Bắt đầu thực thi method: {}.{}",
                joinPoint.getSignature().getDeclaringTypeName(),
                joinPoint.getSignature().getName()
        );
    }

    @After("execution(* com.example.demo.service..*(..))")
    public void logAfterServiceMethod(JoinPoint joinPoint) {
        Long start = startTime.get();

        if (start != null) {
            long executionTime = System.currentTimeMillis() - start;

            log.info(
                    "Kết thúc method: {}.{} | Thời gian thực thi: {} ms",
                    joinPoint.getSignature().getDeclaringTypeName(),
                    joinPoint.getSignature().getName(),
                    executionTime
            );

            startTime.remove();
        }
    }
}
```

## 4.3. Ví dụ Service được ghi log tự động

```java
package com.example.demo.service;

import org.springframework.stereotype.Service;

@Service
public class UserService {

    public String getUserById(Long id) {
        // Giả lập xử lý nghiệp vụ
        return "User ID: " + id;
    }

    public void createUser(String username) {
        // Giả lập tạo user
        System.out.println("Creating user: " + username);
    }
}
```

# 5. Giải thích các thuật ngữ chính trong đoạn code

| Thuật ngữ        | Ý nghĩa                                                |
| ---------------- | ------------------------------------------------------ |
| `@Aspect`        | Đánh dấu class này là một Aspect, chứa logic cắt ngang |
| `@Component`     | Đưa Aspect vào Spring Container để Spring quản lý      |
| `@Before`        | Advice chạy trước khi method mục tiêu được gọi         |
| `@After`         | Advice chạy sau khi method mục tiêu kết thúc           |
| `JoinPoint`      | Đại diện cho method đang bị AOP can thiệp              |
| `Pointcut`       | Biểu thức xác định method nào sẽ bị áp dụng AOP        |
| `execution(...)` | Cú pháp chọn các method cần áp dụng Aspect             |
